# Deploying Imladris

## Quickstart (any Linux box)

```bash
git clone https://github.com/sethdf/imladris.git ~/repos/imladris
cd ~/repos/imladris
./deploy.sh --tailscale-key tskey-auth-... --bws-token 0.e377a...
```

That's it. 4 commands, ~15 minutes (mostly Docker image builds on first run).

## What deploy.sh does

1. Installs Docker + Compose plugin
2. Installs Tailscale + authenticates
3. Installs Bun, Claude Code, BWS CLI
4. Clones repo (if not already present)
5. Sets up PAI symlinks (modules/pai/ → ~/.claude/)
6. Generates .env with random secrets
7. Builds custom Postgres image (pgvector + pgml + AGE)
8. Starts all services via docker compose
9. Sets up pai-sync daemon (systemd)

| Target | Use Case | Notes |
|--------|----------|-------|
| **AWS EC2** | Work | Optional: use CloudFormation for VPC/IAM provisioning first |

## AWS-Specific (Optional)

For AWS deployments, you can optionally use CloudFormation first:

```bash
# Provision EC2 + VPC + IAM (optional — only for AWS)
aws cloudformation deploy \
  --template-file cloudformation/imladris-stack.yaml \
  --stack-name imladris \
  --capabilities CAPABILITY_IAM \
  --parameter-overrides InstanceType=m7id.xlarge

# Then SSH in and run deploy.sh
ssh ec2-user@<instance-ip>
cd ~/repos/imladris && ./deploy.sh
```

Or configure UserData to run deploy.sh automatically on launch.

## Architecture

```
HOST (the box):
  ├── Docker          — container runtime
  ├── Tailscale       — VPN / SSH access
  ├── Claude Code     — AI assistant (runs on host, NOT containerized)
  ├── Bun             — TypeScript runtime for hooks + scripts
  ├── BWS CLI         — secrets from Bitwarden
  └── pai-sync        — systemd daemon, syncs ~/.claude/ → Postgres

CONTAINERS (docker compose):
  ├── windmill_server      — automation platform UI + API
  ├── windmill_worker ×4   — bun script execution
  ├── windmill_worker_native — native worker (PAI filesystem tools)
  ├── windmill_db          — Postgres 16 (pgvector + pgml + AGE + pg_cron)
  └── windmill_lsp         — language server for editor

OPTIONAL CONTAINERS (profiles):
  ├── litellm              — LLM proxy (--profile proxy)
  └── status-dashboard     — health dashboard (--profile dashboard)
```

## Secrets

Secrets are loaded from BWS (Bitwarden Secrets Manager) or a `.env` file:

- `WINDMILL_DB_PASSWORD` — Postgres password (auto-generated by deploy.sh)
- `WINDMILL_ADMIN_SECRET` — Windmill admin token (auto-generated)
- Tailscale auth key — provided via `--tailscale-key` or pre-authenticated
- BWS access token — provided via `--bws-token` or `BWS_ACCESS_TOKEN` env var

## Re-running

deploy.sh is idempotent. Re-run any time to:
- Pull latest repo changes
- Rebuild Postgres image if Dockerfile changed
- Restart services
- Re-link PAI symlinks

## Ansible (Optional)

Ansible roles are still available for advanced host configuration:
- Voice server/client/listener (PipeWire audio, ElevenLabs TTS)
- NVMe instance store optimization (Docker data-root on NVMe)
- Steampipe AWS plugin configuration

These are NOT required for basic deployment. deploy.sh covers 80% of use cases.
